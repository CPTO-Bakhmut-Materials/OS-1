# Вступ (Intro)

PowerShell — це оболонка командного рядка та мова сценаріїв. Це означає, що ми можемо запускати окремі команди в терміналі, а також створювати та виконувати складніші скрипти. Обидва варіанти однаково корисні під час автоматизації завдань адміністрування комп'ютера.

Хоча більшість комп'ютерних інтерфейсів використовують графічний інтерфейс користувача (GUI), розробники та адміністратори все ще використовують інструменти командного рядка, як-от PowerShell, оскільки вони краще справляються з певними завданнями.

Наприклад, базове управління файловою системою: створення папок для 20 користувачів може вимагати десятків або сотень рухів миші та кліків у графічному інтерфейсі. Натомість одна команда в терміналі PowerShell може взяти список імен і створити папку для кожного. Цей приклад стає ще переконливішим, коли кількість користувачів вимірюється тисячами, а завдання для кожного стають складнішими. Хоча графічні інтерфейси можна запрограмувати на виконання будь-якої групи дій, досягти цих цілей за допомогою PowerShell швидше та дешевше.

Використання виводу термінала є основним способом отримання зворотного зв'язку від команд і скриптів PowerShell. Спробуйте ввести таку команду в командний рядок і натиснути Enter:

```powershell
Write-Host "Hello, World!"
```

# Команди (Commands)

Виконання команд у PowerShell допомагає вирішувати багато різних завдань. Команди PowerShell називаються CmdLets (вимовляється як "команд-лети"). Їхні назви допомагають зрозуміти їхню дію, оскільки вони мають формат "Дієслово"-"Іменник", наприклад:

```powershell
Get-Date
```

Наведена вище команда має дієслово `Get` (Отримати) та іменник `Date` (Дата), що означає отримання поточної дати та її виведення в термінал.

У попередній вправі ви вивели "Hello, World!" у термінал за допомогою:

```powershell
Write-Host "Hello, World!"
```

Ця команда виконує запис до хоста, виводячи заданий аргумент у термінал. У цьому випадку рядок "Hello, World!" використовується як аргумент для `Write-Host`. Як ми побачимо згодом, `Write-Host` використовується в скриптах для створення виводу для користувача та відображення даних, що обробляються.

`Get-Date` та `Write-Host` — це лише кілька з багатьох командлетів, які доступні в PowerShell. Щоб дізнатися про інші доступні команди, скористайтеся:

```powershell
Get-Command
```

Без аргументів `Get-Command` виведе всі доступні командлети. Ми також можемо шукати конкретні командлети. Щоб вивести всі команди Get, потрібно передати аргумент за допомогою прапорця `-Verb`:

```powershell
Get-Command -Verb Get
```

Прапорець `-Verb` шукає всі команди, чиє дієслово збігається із заданим аргументом (`Get`).

Ми також можемо шукати за конкретними іменниками:

```powershell
Get-Command -Noun Host
```

Ця команда виводить усі команди, які діють на `Host` (термінал).

Нарешті, як і в будь-якому терміналі, екран може заповнитися великою кількістю тексту. Щоб очистити термінал, виконайте команду:

```powershell
Clear-Host
```

# Використання термінала (Using the Terminal)

Під час роботи в терміналі PowerShell кілька практичних дій можуть полегшити виконання команд.

## Автодоповнення команд (Command Completion)

* Якщо ви введете лише кілька літер команди, а потім натиснете клавішу `Tab`, термінал спробує автоматично доповнити команду.
* Якщо існує лише одна команда, яка відповідає введеному фрагменту (наприклад, `Get-D`), натискання `Tab` автоматично заповнить решту команди як `Get-date`.
* Якщо фрагменту відповідає більше однієї команди (наприклад, `Get-H`), натискання `Tab` виведе список можливих команд:

```text
Get-Help   Get-History   Get-Host
```

Це допоможе визначити, яку саме команду потрібно використати.

## Історія команд (Command History)

Як і в більшості сучасних терміналів, історія команд зберігається у файлі, і до неї можна отримати доступ за допомогою:

```powershell
Get-History
```

Ця команда корисна, якщо ви забули, як раніше виконали завдання, і хочете переглянути минулі дії.

## Навігація історією за допомогою клавіш "Вгору/Вниз" (History with Up/Down Keys)

* Якщо є довгі команди або команди, які ви плануєте часто використовувати, для навігації історією можна використовувати клавіші "Вгору" (`↑`) та "Вниз" (`↓`).
* Почніть з клавіші зі стрілкою вгору, щоб гортати від новіших команд до старіших.
* Ви можете повернутися до новіших команд за допомогою клавіші зі стрілкою вниз.

Такі швидкі комбінації клавіш полегшують навігацію текстовим середовищем PowerShell, дозволяючи зосередитися на головному завданні.

# Написання скриптів (Writing Scripts)

Правильно виконані команди PowerShell можуть багато чого зробити, але іноді нам потрібно виконати складні завдання, що вимагають кількох команд. У таких випадках ми можемо написати скрипт PowerShell і запускати його в терміналі за потреби.

Щоб створити скрипт, відкрийте будь-який текстовий редактор, напишіть команди та збережіть файл із розширенням `.ps1`.

Наприклад, такі команди можна помістити у файл скрипта під назвою `host-commands.ps1`:

```powershell
Write-Host "All commands that act on Host:" 
Get-Command -Noun Host
```

Маючи ці дві команди у файлі `host-commands.ps1`, ми можемо запустити наш скрипт у терміналі за допомогою такої команди:

```powershell
.\host-commands.ps1
```

Це запустить скрипт і видасть наступний результат:

```text
All commands that act on Host:

CommandType     Name
-----------     ----                                   
Function        Clear-Host                             
Cmdlet          Get-Host                               
Cmdlet          Out-Host                               
Cmdlet          Read-Host                              
Cmdlet          Write-Host 
```

Запис `.\` перед назвою скрипта вказує оболонці шукати файл у поточному каталозі та запустити його. У більшості оболонок також можна використовувати прямий слеш `./`.
